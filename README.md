# CohortWeave Open Evaluation

Open, non-sensitive evaluation and reproducibility components for **CohortWeave: Clinically Constrained Generative Enhancement of Fair Cancer-Imaging Cohorts**.

> **Evidence boundary:** this repository contains Spark-phase engineering evidence based on surrogate tabular data, synthetic CT phantoms and a small patient-held-out public real-pixel restoration proxy. It does **not** contain protected EUCAIM data and does **not** demonstrate paired same-patient missing-acquisition completion, clinical validity or achievement of the official ICR ≥95% target. Those remain ADVANCE objectives.

## Ten-minute verification

```bash
git clone <repository-url>
cd cohortweave-open-evaluation
python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate
python -m pip install -r requirements.txt
pytest
python scripts/verify_frozen_results.py
python scripts/check_public_release.py
```

The quick path verifies metric implementations, synthetic fixtures, frozen-result gates and the public-release audit. Full re-execution of the synthetic Spark experiments is available through `python scripts/reproduce_spark.py`. The optional public-DICOM restoration proxy requires `requirements-real-ct.txt` and is slower.

## What is included

- reproducible surrogate tabular synthesis, completion, constraints and fairness experiments;
- synthetic CT phantom harmonisation and radiomics robustness evaluation;
- optional patient-held-out public CT restoration proxy using redistributable package fixtures fetched through dependencies;
- synthetic CSV and NIfTI fixtures with hashes;
- frozen machine-readable Spark results and pseudonymised case-level metrics;
- unit tests, GitHub Actions CI, metric definitions, data provenance and Responsible AI documentation;
- automated audit blocking common medical files, model weights, application documents, partner letters and obvious secrets.

## What is deliberately excluded

Protected clinical JSON and DICOM studies, raw challenge archives/indexes, patient-level exports, private model weights/checkpoints, `.pkl` cohorts, infrastructure credentials/endpoints, application documents, CVs, Letters of Intent and proprietary deployment components.

## Headline Spark evidence

- surrogate continuous KS: approximately 0.017–0.018;
- sex ratio: 2.135:1 → 1.0:1 on the synthetic cohort;
- PD-L1 completion MAE improvement: 32.2% versus median imputation;
- four-rule contradiction rate: 67.0% → 0% with the transparent constraint-repair proxy;
- phantom kVp-like Wasserstein reduction: 94.0%, with Joint-Entropy CCC 0.9879;
- public real-pixel proxy: five patient IDs, six frames, 36 corruption cases/method; TV mean PSNR 32.93 dB and SSIM 0.833;
- illustrative surrogate fairness stress test: female C-index +0.0715 and sex-gap reduction 36.3%.

See [`docs/spark_results.md`](docs/spark_results.md) for scope, uncertainty and limitations.
